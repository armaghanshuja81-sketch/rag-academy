🗄️ Relational Databases & SQL

Databases store your app's data permanently. SQL is the language you use to talk to them.

What is a Relational Database?

Think of it as a set of Excel spreadsheets that your app can read and write programmatically.

  • Table — Like a spreadsheet tab (e.g., "Users" table, "Orders" table)
  • Row — A single record (e.g., one user's data)
  • Column — A field (e.g., name, email, age)
  • Primary Key — A unique ID for each row (usually id)
  • Foreign Key — Links rows across tables (e.g., order.user_id → users.id)

The 4 SQL Operations (CRUD)

Every database operation is one of these four:

📝 CREATE (INSERT)

INSERT INTO users (name, email)
VALUES ('Armaghan', 'a@b.com');

📖 READ (SELECT)

SELECT * FROM users
WHERE email = 'a@b.com';

✏️ UPDATE

UPDATE users SET name = 'New'
WHERE id = 1;

🗑️ DELETE

DELETE FROM users
WHERE id = 1;

SQLite vs PostgreSQL

SQLite (Learning)PostgreSQL (Production)
Single file on diskRuns as a server
Built into PythonNeed to install & configure
One user at a timeThousands of concurrent users
Great for learning & devUsed in real applications
KEY INSIGHT: The SQL is almost IDENTICAL. Switch from SQLite to PostgreSQL by changing only the connection code. All your queries stay the same.

Flask + SQLite Example

Python 
import sqlite3

# Connect (creates file if not exists)
conn = sqlite3.connect("myapp.db")
cursor = conn.cursor()

# Create table
cursor.execute("""
    CREATE TABLE IF NOT EXISTS users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        email TEXT UNIQUE
    )
""")
conn.commit()

# Insert data (⚠️ ALWAYS use ? placeholders!)
cursor.execute(
    "INSERT INTO users (name, email) VALUES (?, ?)",
    ("Armaghan", "armaghan@email.com")
)
conn.commit()

# Query data
cursor.execute("SELECT * FROM users")
rows = cursor.fetchall()

# Close
conn.close()
🚨 SECURITY: NEVER do this!
f"INSERT INTO users VALUES ('{user_input}')" ← SQL injection!
Always use ? placeholders!

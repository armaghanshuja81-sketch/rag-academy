The RAG Architecture

RAG = Retrieval-Augmented Generation. It has three steps:

Step 1: RETRIEVE

When a user asks a question, search your documents for relevant information.

User Question → Convert to Embedding
→ Search Vector Database
→ Find Top 3 Most Relevant Chunks

Step 2: AUGMENT

Add the retrieved information to the prompt sent to the LLM.

System: Answer based ONLY on this context:

Context:
--- Chunk 1 ---
...document text...
--- Chunk 2 ---
...more text...

User: What is RAG?

Step 3: GENERATE

The LLM generates an answer using ONLY the provided context.

LLM Response: "RAG stands for Retrieval-Augmented Generation,
which is a technique that combines information retrieval
with text generation..."

Code (No Frameworks)

def rag_pipeline(question, documents):
    # 1. Chunk documents into pieces
    chunks = split_into_chunks(documents)

    # 2. Embed all chunks and the question
    chunk_vecs = [embed(c) for c in chunks]
    question_vec = embed(question)

    # 3. Find most similar chunks
    similar = find_top_k(question_vec, chunk_vecs, k=3)

    # 4. Build prompt with context
    context = "\n".join(similar)
    prompt = f"Context:\n{context}\n\nQuestion: {question}\nAnswer:"

    # 5. Get answer from LLM
    answer = llm_generate(prompt)
    return answer

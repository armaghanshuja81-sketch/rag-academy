🤖 RAG Demo — See It Work

RAG = Retrieval-Augmented Generation. Ask a question about our sample documents. The system will: (1) Search for relevant info, (2) Show you what it found, (3) Generate an answer. This is how ChatGPT-like apps use YOUR data.

📖 What is RAG?

LLMs (like ChatGPT) only know what they were trained on. If you ask about YOUR company's documents, it doesn't know the answer.

RAG solves this:

  1. Retrieve: Search your documents for relevant info
  2. Augment: Add that info to the prompt
  3. Generate: LLM answers using your document's content

Python (Flask Route) 
@app.route('/rag-demo', methods=['GET', 'POST'])
def rag_demo():
    if request.method == 'POST':
        query = request.form['query']

        # STEP 1: RETRIEVE — Find relevant chunks
        all_chunks = get_all_chunks()
        scored = []
        for chunk in all_chunks:
            score = simple_keyword_match(query, chunk['text'])
            scored.append((score, chunk))
        scored.sort(reverse=True)
        top_chunks = scored[:3]  # Top 3 matches

        # STEP 2: AUGMENT — Build the prompt with context
        context = "\n".join([c['text'] for _, c in top_chunks])
        prompt = f"Context:\n{context}\n\nQuestion: {query}\nAnswer:"

        # STEP 3: GENERATE — Get answer (real: call LLM API)
        answer = generate_answer(prompt, top_chunks)

        # Save to database for history
        save_to_db(query, answer)

        return render_template('rag_demo.html',
            result={'chunks': chunks, 'answer': answer},
            query=query, documents=documents)

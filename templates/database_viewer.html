📖 What is a Database?

A database is like a set of Excel spreadsheets that your app can read and write. Each "table" is like a sheet, each "row" is a record, each "column" is a field.

This app's database has these tables:

  • lesson_progress — tracks which lessons you've completed
  • user_queries — stores RAG demo questions
  • database_lessons — sample data for learning SQL

🔗 SQLite vs PostgreSQL

SQLite (this app): A file on your computer. Perfect for learning. No server needed. Built into Python.

PostgreSQL (real apps): A database server. Multiple users can access it at once. Used in production. You'd deploy it on Render, Railway, or Supabase.

KEY INSIGHT: SQL is the SAME for both. Only the connection changes. Learn SQL once, use it everywhere.

🔬 What Just Happened?

When you clicked "Run Query", this happened:

Python (Flask) 
# 1. Flask receives your SQL query from the form
query = request.form['query']

# 2. Flask opens a connection to the database
conn = sqlite3.connect('rag_academy.db')
conn.row_factory = sqlite3.Row
cursor = conn.cursor()

# 3. Flask executes your SQL
start = time.time()
cursor.execute(query)
results = cursor.fetchall()
elapsed = time.time() - start

# 4. Flask gets the column names from cursor.description
columns = [desc[0] for desc in cursor.description]

# 5. Flask converts rows to dictionaries
rows = [dict(row) for row in results]

# 6. Flask closes the connection
conn.close()

# 7. Flask renders this template with the results
return render_template('database_viewer.html',
    tables=tables, query_result={
        'columns': columns,
        'rows': rows,
        'time': elapsed
    })

This is the complete data flow: Your query → HTTP POST → Flask route → SQLite query → Results → HTML response → You see the table.

## 3. Dark Mode Overrides

**Token layer**: 20 tokens redefined under `[data-theme="dark"]` — shadows, surfaces, text, accent. This is good.

**Per-element overrides** (4 scattered):
- `[data-theme="dark"] .clay-nav` — nav background + box-shadow
- `[data-theme="dark"] pre` — background + color
- `[data-theme="dark"] .clay-playground__editor textarea` — background
- `[data-theme="dark"] .clay-hero h1` — gradient text

**Missing from dark overrides**: tier cards, module cards, quiz options, flow boxes, RAG sources, trace cards, code blocks, alerts, badges. These rely on token-level changes only (mostly OK since they consume tokens).

## 6. Accessibility Issues

- **Focus rings**: Uses `:focus` not `:focus-visible` on inputs
- **Color-only indicators**: Quiz correct/incorrect has both color + icon ✓, but progress bars, badges, and tier dots rely on color alone
- **Touch targets**: Theme toggle (40px) passes. Nav links OK. Quiz option letter circles (28px) are borderline.
- **Contrast concerns**: `--clay-text-muted` (#5a688a) on `--clay-surface` (#f4f7fc) = ~4.3:1 — borderline for AA small text. Dark mode: `--clay-text-muted` (#8b9dc0) on surface (#162032) = ~6.1:1 — passes.

## 7. Responsive Issues

- Single breakpoint at 768px — no tablet (1024px) or large desktop treatment
- `.clay-grid-2/3/4` collapse to 1 column at 768px — fine for mobile, but 4-col on tablet is cramped
- `.clay-lesson-sidebar` drops to static at 768px — good
- No logical CSS properties — all physical (`margin-left`, `padding-right`, etc. hardcoded)
- The Lessons page `.clay-grid-2` lesson cards at 50% width per card may be tight on tablet

## Summary: What To Change Per Phase

**Phase 0 (token VALUES)**:
- Kill dual shadows → single subtle shadow (or border)
- Kill bounce easing → standard ease-out
- Kill inflated radii → 6/10/12
- Swap Fredoka → serious grotesk (e.g. Inter)
- Fix 18+ hardcoded colors → tokens
- Standardize spacing on 8px scale

**Phase 1 (dark mode)**:
- Consolidate 4 scattered overrides into token layer
- Add missing dark tokens for code blocks, flow boxes

**Phase 2 (components)**:
- Remove inset shadows from inputs, progress, inset cards
- Remove hover bounce/translate from cards and buttons
- Remove decorative left-border strips (blockquote, quiz review, RAG sources, trace cards)
- Standardize focus rings on `:focus-visible`
